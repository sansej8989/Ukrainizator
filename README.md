# Українізатор v5.0.0

Скрипт для PowerShell, який автоматично українізує Windows 10/11 в один прохід: встановлює мовний пакет, перемикає інтерфейс і регіональні стандарти на uk-UA, чистить російську мову/розкладку/додаткові компоненти та наводить лад у дрібних системних дрібницях. Розрахований на реальне, повторюване, персональне використання — а не одноразовий демо-скрипт.

## Швидкий старт

Найпростіше — через готові `.bat`-лаунчери (самі підіймають права адміністратора, окремо просити «Запуск від імені» не потрібно):

| Файл | Що робить |
|---|---|
| `Start_MANUAL.bat` | Інтерактивний режим: скрипт питає підтвердження на ключових кроках (режим встановлення, перезавантаження). |
| `Start_AUTO.bat` | Тихий unattended-режим: без жодних запитань, для всіх користувачів, з автоматичним перезавантаженням (5-секундне вікно на скасування). |

Або напряму через PowerShell (від імені адміністратора):

```powershell
.\ukrainizator.ps1                                    # інтерактивний режим
.\ukrainizator.ps1 -Silent -Mode All -NoRebootPrompt   # тихий unattended-режим
```

**Вимоги:** Windows 10/11, PowerShell 5.1+, права адміністратора.

## Що саме робить скрипт (12 кроків)

| № | Крок | Опис |
|---|---|---|
| 1 | Перевірка прав адміністратора | Без прав адміністратора скрипт коректно зупиняється з поясненням. |
| 2 | Перевірка версії PowerShell | Потрібен 5.1+; заодно визначається конкретна збірка/редакція Windows (для кроку 6-7). |
| 3 | Підтвердження | Показує попередження про зміни системи, питає продовжити (пропускається в `-Silent`). |
| 4 | Точка відновлення | Створює точку відновлення системи перед втручанням у реєстр (поважає `-WhatIf`). |
| 5 | Вибір режиму встановлення | Для всіх користувачів системи чи лише поточного (`-Mode All` / `-Mode Current`). |
| 6 | Перевірка системних модулів | Визначає, чи доступний сучасний `Install-Language`, чи потрібен запасний DISM-шлях. |
| 7 | Встановлення мовного пакету | Ставить uk-UA двома можливими шляхами — див. [Сумісність зі збірками](#сумісність-зі-збірками-windows). |
| 8 | Мова інтерфейсу | `Set-WinUILanguageOverride` + реєстр + копіювання налаштувань на екран привітання/нових користувачів. |
| 9 | Регіональні формати | Культура, системна локаль, розташування, **часовий пояс (Київ)**, **перший день тижня — понеділок**, **символ валюти ₴**. |
| 10 | Дерусифікація та розкладки | Прибирає ru-* мову/розкладку з профілю; **видаляє додаткові ru-компоненти** (розпізнавання мовлення, рукописне введення, OCR); **чистить кеш підказок набору тексту**. |
| 11 | Оптимізація та перезапуск провідника | Дрібні реєстрові оптимізації, очищення кешу шрифтів, перезапуск `explorer.exe`. |
| 12 | Перевірка результату | Повторно опитує систему (локаль, мову, розкладки, регіон) і показує PASS/FAIL по кожному пункту — а не просто вірить, що команди відпрацювали. |

Кожен крок, крім 1-3, за замовчуванням **не зупиняє скрипт при помилці** — позначається як пропущений/частковий, фіксується у підсумку, виконання йде далі (див. [Продовження після помилки](#продовження-після-помилки)).

## Параметри командного рядка

| Параметр | Тип | Опис |
|---|---|---|
| `-Silent` | switch | Без інтерактивних запитань. |
| `-Mode` | `All` \| `Current` | Застосувати до всіх користувачів чи лише поточного. За замовчуванням `All`. |
| `-NoRebootPrompt` | switch | Не питати про перезавантаження — автоматичний рестарт (з 5-секундним вікном на скасування). |
| `-Force` | switch | Пропустити швидку перевірку «вже налаштовано» і прогнати все заново. |
| `-Revert` | switch | Відкотити налаштування з останнього збереженого знімку. |
| `-WhatIf` | switch | Показати, що БУЛО Б зроблено, нічого реально не змінюючи. |
| `-ComputerName` | string[] | Список віддалених машин — запустити скрипт там через PS Remoting замість локально. |
| `-Credential` | PSCredential | Обліковий запис для віддаленого запуску разом із `-ComputerName`. |

Приклади:

```powershell
# Попередній перегляд без змін
.\ukrainizator.ps1 -WhatIf

# Примусовий повторний прогін, навіть якщо все вже налаштовано
.\ukrainizator.ps1 -Force

# Відкат до стану "до запуску"
.\ukrainizator.ps1 -Revert

# Розгортання на трьох машинах одночасно
.\ukrainizator.ps1 -ComputerName PC1,PC2,PC3 -Credential (Get-Credential)
```

## Живий інтерфейс (ANSI-рушій)

Скрипт має власний легкий UI-рушій без залежностей:

- Якщо термінал підтримує ANSI/VT (сучасний Windows Terminal, PowerShell 5.1+ на Windows 10/11) — показує рамку, кольорові іконки статусу (✓ / ✗ / ●), спінер, **прогрес-бар із плавним градієнтом блакитний → золотий** (у стилі прапора), і перемальовує кадр повністю при кожній зміні (без крихкого «підняти курсор на N рядків», яке ламалося при неточній ширині вікна на старті).
- Якщо ANSI недоступний (перенаправлений вивід, нетипові хости) — автоматичний резервний **лінійний режим**: кольорові рядки з вирівнюванням крапками-заповнювачами, без ANSI-кодів, теж охайний, просто без анімації.

Перемикання відбувається автоматично, без участі користувача.

## Ідемпотентність і швидка перевірка

На старті (якщо не задано `-Force` і не `-WhatIf`) скрипт перевіряє, чи система вже повністю українізована (системна локаль, мова інтерфейсу, розкладки, регіон). Якщо так:

- у `-Silent` — одразу виходить, нічого не чіпаючи;
- в інтерактивному режимі — питає, чи повторити прогін ще раз (за замовчуванням «ні»).

Це робить повторні запуски (наприклад, після кожного великого оновлення Windows) швидкими й безпечними — жодного зайвого ризику для вже коректно налаштованої системи.

## Відкат змін (`-Revert`)

Перед будь-якими реальними змінами (і якщо не `-WhatIf`) скрипт зберігає JSON-знімок поточного стану — `Ukrainizator_backup_ГГГГММДД_ГГХХСС.json` у корені теки скрипта: системну локаль, мову інтерфейсу, культуру, розташування, список мов. Зберігаються останні 5 знімків.

```powershell
.\ukrainizator.ps1 -Revert
```

прочитає найновіший знімок, покаже, що саме буде відновлено, запитає підтвердження і поверне систему до попереднього стану — швидше й простіше, ніж відкочувати через повну точку відновлення системи.

## Продовження після помилки

Кроки 1-3 (права адміністратора, версія PowerShell, згода користувача) — справжні блокери: без них продовжувати немає сенсу, скрипт коректно зупиняється.

Усі інші кроки (4, 6-12) при помилці **не зупиняють весь прогін** — позначаються як пропущені/часткові, причина записується через `Add-StepIssue`, виконання йде далі. Наприкінці, якщо були якісь зауваження, показується їхній підсумок окремою панеллю (з людяним описом причини — див. нижче) замість того, щоб просто мовчки завершитись на середині 11 кроку.

## Захист від паралельного запуску

Скрипт створює іменований м'ютекс ОС (`Global\UkrainizatorRunning`) на самому старті. Якщо його вже утримує інший запущений екземпляр (випадковий подвійний клік, забутий інстанс у фоні) — новий запуск одразу побачить це, покаже попередження і коректно вийде, замість того, щоб два процеси одночасно писали в реєстр і калічили налаштування один одному. М'ютекс звільняється явно наприкінці; якщо процес завершиться аварійно — ОС звільнить його сама, тож «зависань» замка не буває.

## Сумісність зі збірками Windows

Крок 2 визначає й логує конкретну збірку/редакцію Windows (`CurrentBuildNumber`, `DisplayVersion`, `ProductName`).

Крок 6-7 обирає спосіб встановлення мовного пакету залежно від того, що реально доступне на цій системі:

1. **Сучасний шлях** — `Install-Language` / `Get-InstalledLanguage` (модуль `LanguagePackManagement`), якщо він є.
2. **Запасний шлях (DISM)** — `Add-/Get-WindowsCapability` (`Language.Basic~~~uk-UA~0.0.1.0` +, де доступно, шрифти/мовлення/рукопис/OCR), якщо сучасного API немає. Ці командлети значно старіші й практично гарантовано присутні на будь-якій підтримуваній збірці Windows 10/11.

Якщо жоден із двох шляхів недоступний — крок чесно позначається як пропущений з поясненням причини, а не провалює весь скрипт.

Ще одна дрібниця для надійності: усі типові технічні `.NET`-помилки (немає інтернету, відмовлено в доступі, команда відсутня в цій збірці, шлях не знайдено, брак місця на диску) автоматично перекладаються на зрозумілу українську через `Get-FriendlyErrorMessage`, з оригіналом у дужках для діагностики.

## Перевірка цілісності скрипта

Поруч зі скриптом опційно може лежати файл `ukrainizator.ps1.sha256` з очікуваним SHA-256 хешем. Якщо він є і не збігається з фактичним вмістом файлу — скрипт голосно попередить (можливе пошкодження чи стороннє втручання) і запитає явне підтвердження продовжити.

Згенерувати/оновити хеш після легітимного редагування:

```powershell
(Get-FileHash -Path .\ukrainizator.ps1 -Algorithm SHA256).Hash | Set-Content .\ukrainizator.ps1.sha256
```

> Це не заміна справжнього Authenticode-підпису (який вимагає сертифіката — платного від довіреного центру, або самопідписаного, якому довіряють лише ті машини, куди його вручну імпортували). Це практичний захист від випадкового пошкодження файлу, не від цілеспрямованого зловмисника з доступом на запис у ту саму теку.

## Локалізація

Тексти інтерфейсу лежать у `locales/uk-UA.json` та `locales/en-US.json` (99 ключів кожен, синхронізовані). Скрипт визначає мову системи автоматично; якщо ключа немає у файлі — замість сирого ідентифікатора показується людяний текст-заглушка (перше слово з великої літери, підкреслення замінені на пробіли).

## Логи

Кожен запуск пише детальний лог у `log\Ukrainizator_ГГГГММДД_ГГХХСС.log`. Перед кожним новим запуском тека автоматично чиститься до 3 найновіших файлів — теку скрипта це не засмічує.

## Віддалений запуск на кілька машин

```powershell
.\ukrainizator.ps1 -ComputerName PC1,PC2,PC3 -Credential (Get-Credential)
```

Локальна машина в цьому режимі нічого сама не змінює — копіює вміст скрипта на кожну вказану машину через PS Remoting (`Invoke-Command`) і запускає його там у тихому режимі. Потребує увімкненого PS Remoting (`Enable-PSRemoting`) на цільових машинах і мережевого доступу до них.

## Відомі обмеження

- **Windows Home** офіційно не підтримує повноцінну зміну мови інтерфейсу через Language Packs — це обмеження самої Microsoft, не скрипта.
- **Групові політики (GPO)** можуть відкотити мовні/регіональні налаштування назад після наступної синхронізації — скрипт наразі не перевіряє це заздалегідь (актуально лише для доменних машин).
- Реєстрові тонкощі Office-спелчекера свідомо **не** зачіпаються — надто версійно-залежні й ненадійні, щоб гарантувати коректну роботу.
- Перевірка цілісності через SHA-256 — захист від пошкодження файлу, не повноцінна заміна Authenticode-підпису.

## Історія версій

- **v5.0.0** — виправлено неапробовані дієслова командлетів (`Build-Frame`→`New-Frame`, `Render-UI`→`Show-Frame`); ротація логів у `log/` (3 останні); людяні повідомлення про типові помилки; захист від паралельного запуску (м'ютекс); універсальна сумісність зі збірками Windows через DISM-запасний шлях.
- **v4.2.0** — 24-бітна кольорова палітра, градієнтний прогрес-бар, двоколірний заголовок.
- **v4.1.0** — сучасний ANSI/VT UI-рушій (рамка, спінер, живе перемальовування) з автоматичним резервним лінійним режимом.
- **v4.0.x** — базова функціональність: 11 кроків українізації, локалізація, звітність.
